How a Website Opens in a Web Browser

Every time you type a URL and press Enter, five stages happen behind the scenes — usually in under 3 seconds.

1. DNS — Find the Address

When you type www.google.com, the browser does not understand domain names. It contacts a DNS (Domain Name System) server and asks: "What is the IP address of this domain?"

The DNS server replies with something like 142.250.190.46. Now the browser knows exactly where to connect.

Think of DNS like a phonebook — you search by name, it gives you the number.

2. TCP — Open the Pipe

The browser now connects to the server at that IP address using TCP (Transmission Control Protocol). Before any data is sent, a 3-way handshake happens:

  • Browser → Server: "Can I connect?" (SYN)
  • Server → Browser: "Yes, I am ready!" (SYN-ACK)
  • Browser → Server: "Great, let's go!" (ACK)

After this, a reliable connection is established between the browser and the server.

3. TLS — Make it Secret

If the website uses HTTPS, a TLS (Transport Layer Security) handshake happens next. The browser and server agree on a secret encryption key.

All data sent after this is encrypted — even if someone intercepts it, they will only see random gibberish. This is why HTTPS is safe and HTTP is not.

The lock icon you see in the browser address bar means TLS is active.

4. HTTP — Ask and Answer

Now the browser sends an HTTP Request to the server, asking for the webpage:

  • Method: GET (asking for the page)
  • Path: / (which page to load)
  • Headers: browser info, language, cookies etc.

The server receives this request. If it is a dynamic page, it queries a database, builds the HTML, and sends back an HTTP Response with the page content and a status code like 200 OK.

404 means page not found. 200 means everything is fine.

5. Render — Draw the Page

The browser now has the HTML. It goes through these steps to show you the page:

  • Reads HTML and builds the DOM (structure of the page)
  • Fetches CSS files and builds the CSSOM (styles)
  • Runs JavaScript files
  • Combines DOM + CSSOM into a Render Tree
  • Calculates Layout — position and size of every element
  • Finally Paints the pixels on your screen

What you see on screen is the final result of all five stages working together.
